Become a Web Developer:

Full-Stack vs. Front-End

And everything in between.

Getting Started

Types of Web Developers

  • Work with languages that render in the browser (client): HTML, CSS, JavaScript
    • Front-end technologies
  • Back-end development: certain content has to be customized for each user based on their data
    • Most action happening on server
  • Full stack developer - someone who works on both front and back end
  • Web stack - collection of tools used for web development
    • Realistically all developers have to have idea of both front and back
    • Unicorn among developers

The Role of HTML

  • HyperText Markup Language
  • Content and structure of webpage
  • Hypertext means documents link to one another
  • Information architecture (IA): structural design of a website’s navigation
  • Markup language - set of tags used to define structure
    • Defines role each content has
  • Master it
  • DOM - document object model
    • Outline or map of content

The Role of CSS

  • User Experience- UX
  • User Interface - UI
  • Navigation drives user experience
  • Cascading Style Sheets
  • 2 parts
    • Defines set of rules how DOM is translated
    • Priorities how stuff is rendered on page
      • Colors, sizes, fonts
  • CSS Zen Garden
  • Content has to be responsive and adjust to proportions of devices
  • CSS always changing
    • CSS3 has been split into independent modules that will be expanded in the future

The Role of Javascript

  • Responsible for managing interaction with user
  • JS interacts with DOM, handling events, loading and unloading media, and creating modifying and deleting elements in a page
  • New version always on the horizon
  • Have to worry about browser support issues

Back-end Essentials

  • Use databases to manage the flow of information sent to the user for customized user experiences
  • Set of tools used for web development - stack
  • PHP and MySQL most popular back end combination
    • Then try Node.js, Ruby on rails, Python, etc
  • Pays in the beginning to be flexible, pays even more to specialize later on

Servers

  • In charge of managing interaction between data and how data is presented to user
  • Master a server platform
    • Linux - more common and less expensive, supported by active community, fully open source
    • Windows - popular with large companies, some open source projects
  • Good way to get started is to install a local version of a PHP and MySQL server in your own workstation

Server-side Scripting

  • Back-end
  • Manages data on server and converts to information that browser can display
  • Important to have exposure to more than one platform
  • Main server-side programming languages are:
    • NodeJS ← JS as server language
    • PHP ← make you the most marketable
    • ASP.NET
    • Ruby
    • Python
    • .NET (Windows server only)

Database Languages

  • Need to be familiar with language that lets their code query databases for info
  • Most servers use variant of Relational DataBase Management System (RDBMS) called SQL
    • Tables with fields
    • Most popular implementations are:
      • Oracle
      • MySQL
      • PostGresQL
    • Other popular ideas
      • NoSQL
        • Whereas SQL encourages separating data into different tables
        • NoSQL allows data with no specific structure
          • Popular with Google’s Big Table which powers search engine
        • Sometimes these languages are referred to as document stores
      • To learn to create a web application with NoSQL structure, look at AngularJS 1: Building a Data-Driven APP
      • MongoDB, Couch Base, Couch DB
  • Learn at least SQL RDBDMS and a NoSQL platforms

Learning and Training

Train for the Industry

  • 4 paths most people take to become web developers:
    • Instructor-led training
    • Self study
    • Working on projects
    • Learning that only happens as a result of working with real clients
  • Software changes so rapidly that it’s more important to learn the basics and then quickly adjust to new technologies
  • Keys for becoming great developer:
    • Read online documentation
    • Examine other developers’ code
    • Explore new ideas that can make your job more efficient
  • HAVE TO BUILD REAL PROJECTS
    • Make mistakes!

Traditional Education

  • Do I need college degree to get a job?
    • What value does having a college degree bring to my ability to get hired?
      • College grades earn $1m more over lifetime
      • Some companies require college degree
      • By 2020, 65% of jobs will require a degree
    • Less than half of developers have a degree in computer science
  • Coding Bootcamps
    • Hyperfocused
    • Quickly teach practical skills necessary to get a job in the industry
    • Hard, 40+ hour weeks over 9 to 12 weeks
    • BUT 88% get job within 120 days of graduation
    • Teach peer programming, test-driven development, cutting edge tools and techniques
    • Low acceptance rates
    • Great if you are certain you want to get into web development

Other Learning Options

  • Google and Youtube good for quick answers
  • Online groups
    • Slack.com to create communities around a product
    • StackOverflow.com for questions and answers
  • Books
    • More reliable because reviewed by industry professionals
  • Online training
    • Lynda!
    • Free code on GitHub
      • Forking - using premade code to make your own thing
      • CodePen
      • JSBin

Social Learning Opportunities

  • Web development is a team sport
    • Make contacts within industry to learn about local community
      • Meetup.com to find developer events
    • Conferences
      • Some pricey, some are free
      • BarCamp
      • Hack-a-thon

Core Developer Tools

  • 3 tools every developer should master
    • Editor
      • Most contact with
      • Integrated Development Environment (IDE)
        • Take longer to master
        • VisualStudio, NetBeans, Eclipse, Android Studio
      • Text editors
        • Load faster but take less to customize
        • Sublime Text, VIM, Atom
    • Terminal
      • Command Line Interface (CLI)
    • Browser
      • Where you test what you’ve created
      • Debug HTML, CSS, JavaScript
      • Test animations, sync to phones for mobile testing

Other Essential Tools

  • phpMyAdmin - lets you visual data through easier interface than terminal
    • Default with most server providers
  • Version control
    • Time machine for your code
      • Git and GitHub
  • Build tools
    • Let you create workflows for common tasks
    • Transpiling, minifying and debugging
  • Frameworks
    • Libraries of pre-built code that help you build sites faster
      • Bootstrap
      • jQuery
      • AngularJS and React
        • Single page applications and interfaces
      • Wordpress
        • CMS

Core Concepts

  • Accessibility
    • Foundations of UX: Accessibility
  • Agile workflows
    • Strategy of producing software in short, two-week sprints
    • Then teams reassess plans and consider new developments
  • Test-driven development
    • Considers creating tests at same time you create features

Business and Leadership Skills

  • 96% of developers say they work on a team
  • Listen and be willing to try alternative ideas
  • Present and defend ideas
  • Keep LinkedIn and portfolio up to date
    • How to Rock Social Media
  • Learn to Negotiate Salaries

Getting a Job

Types of Jobs

  • Most developers identify as Full Stack
  • 70% of developers are employed full-time globally
  • Only about 15% of developers in US are looking for a job
  • 12% work remotely
    • Higher personal job satisfaction
  • Salary > Work/life balance usually (not me)

Maximize Compensation

  • Lucrative
  • Depends on where you live and specific skills
  • Median wage is $65k in May 2015
  • 27% growth

Land a Job

  • Most jobs happen through referrals
  • Build relationships through recruiters
  • Seek opportunity directly
    • Attend meetups and connect through social media
