Chrome DevTools

How Chrome's built-in Developer Tools makes our lives easier.

Instructions:

Hello Everyone!


Learning Chrome DevTools is a fundamental skill for anyone that wants to get into Web Development. This week I want you all to search YouTube for Chrome DevTools Tutorials. Because everyone is at different level, I would expect to find your tutorials to reflect your knowledge level and THAT is a GOOD thing. So find a video that helps you either understand what Chrome DevTools is or add a new skill to your existing Chrome DevTools skills level. Provide us the link to the tutorial you select and then take a printscreen of at least one of the things you learned from the tutorial, upload to Google Drive and provide link here. Write 150 words about your experience here and reply to 2 of your fellow students post, make sure you replies are substantive.


Have fun!!! This is a great tool!

My response:


I actually have not had much hands-on experience with DevTools, thought I do have a basic understanding of its layout, functionality, and usefulness to developers. The first video I found (that was useful to me) was one by a YouTuber called The Net Ninja. I’m taking client-side scripting in addition to this class and one of the cool things I learned from this tutorial is that the console in DevTools can check your JavaScript syntax and display errors in the console, making it a great debugging tool.



On YouTube I noticed there were very few “DevTools for beginners” overview-type of videos, and a lot of tutorials for more intermediate/advanced users addressing very specific topics. I had more luck using Lynda.com. Learning Chrome Web Developer Tools is a little lengthy at an hour long, but it was only released a little over a month ago and is a good introduction for people at my level. The coolest thing I learned was that you can emulate mobile connections or simulate “throttled” connections in the Network panel. I also learned a handy trick of holding down shift while refreshing the page, which makes Chrome request all assets of the page again instead of using cached versions, which is a critical step if you're measuring the speed at which your browser is downloading assets from a web server.
