Visual Studio Code for Web Developers

Using VS Code to its fullest potential for web development.

Overview of VS Code

The VS Code UI

  • Left hand bar is called view bar
  • Top icon goes to Explorer View (ctrl + shift + e)
  • Magnifying glass goes to Search View (ctrl + shift + s)
  • Integrated Git View (ctrl + shift + g)
  • Debugger View (ctrl + shift + d)
  • Extensions View (ctrl + shift + x)
  • Status bar at the bottom
  • Command palette: ctrl + shift + p
  • Zen mode (ctrl + k + z) to focus just on code you are writing

Basic Editing

  • If you click on line and copy and paste, copies whole line if there’s not selection
  • Indentation organizes code and makes it readable ( ctrl + < and >)
  • Right-click when no selection, format document automatically
  • Ctrl + i = adding a line to a selection

The Integrated Terminal

  • To view terminal: ctrl + `
  • Can open multiple terminals, trash can to close

Integrated Source Control

  • Git --version
  • Git View with number means there are modified files with count
  • Green bar means things have been added, blue means things have changed, and red means things have been deleted

Customizing VS Code

Configuring Preferences

  • Split-code editor
  • Settings is a JSON file
  • Left side is instructions, right side is editor
  • User settings applies to all files and projects, workspace settings applies to specific workspaces and overrides user settings
  • Right click property on left side by clicking on pencil
  • Can set settings dependent on language

Setting Keyboard Shortcuts

  • Can set context for what circumstances key applies
  • Marketplace has extensions for keybinding presents

Using Extensions

  • https://marketplace.visualstudio.com
  • Can view documentation
  • Need to reload VS Code to complete extension installation

Essential Extensions

  • HTML Snippets
  • jQuery Code Snippets
  • Beautify
  • Express
  • REST Client

Working with Snippets

  • Smart fragments of code that can be called just typing a few characters
  • Hitting tab put skeleton of code construct
  • Defined using JSON and can be customized

Color and Icon Themes

  • To choose color scheme, look under Preferences menu
  • Find new themes at Marketplace
  • File icon themes also in Preferences, with more in the Marketplace

Using the Editor

Files, Folders and Projects

  • Files, Folders and Projects
    • Show current files and folders in project, and open editors
    • Can have up to 3 editors open side by side per file at once
    • Ctrl + tab to cycle through all open editors
    • Ctrl + alt + t to close all but open tab, reverse with ctrl + shift + t
  • Side by Side Editing
    • Ctrl + backslash or click up split editor upper right to show two windows of code
    • Ctrl + 1 thru 3 to switch between tabs quickly
  • IntelliSense
    • Code completion, code hinting
    • Can turn it off in settings called quick suggestions
  • Finding and Replacing
    • Ctrl + f
    • Enter key to advance through results
    • Ctrl + shift + f to find through different files

Advanced Editing

Editor Features

  • Remembers content of unsaved files in case you close without saving (hot exit)
  • Setting can be toggled in preferences
  • Can enable auto save under File
  • There’s also a format on save option in preferences
  • Careful if working with others when using auto format because it might mess up versioning
  • Format when paste

Web Code Editing

  • Can quickly comment sections of code by highlighting and using ctrl + forward slash
  • /** makes JSdoc skeleton to comment parameters above JS functions
  • Real time updates in preview window
  • Supports Emmet snippets

Navigating Code

  • Ctrl + g to go to specific line in a file
  • Ctrl + shift + o to go to specific symbols
