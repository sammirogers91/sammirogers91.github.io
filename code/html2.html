HTML Essential Training: Part II

A look at structuring content and creating links.

Structuring Content

The Value of Structure

  • Semantics - structuring in a way that illustrates meaning of the content
  • There are people using assistive technology and machines, so actual structure of page needs to convey meaning of content

Controlling Document Outlines

  • Important
  • HTML5 has new elements: nav, article, section, aside
  • Aside - related to content on page but not necessary in order to complete list
  • Created through HTML5 outlining algorithm

The Nav Element

  • https://w3.org/TR/html5
  • If you have multiple links inside nav element, should order them in a list Intended for major navigation blocks only
  • Screenreader can skip over it or go directly to it
  • Isn’t really needed for groups of links that aren’t navigational links that link to things within your own site

The Article Element

  • Forum posts, blog entries, etc
  • Think of it as article of clothing - individual elements that can stand alone
  • Only use if content can stand alone or be syndicated

The Section Element

  • Generic section of document or application
  • Thematic group of content with a heading
  • Chapters, tabbed pages, numbered sections of a thesis, etc

The Aside Element

  • Designed to represent related content
  • Represented as sidebars in printed typography
  • Tangentially related content
  • Not absolutely necessary but related, info that might be replaced or deleted at some point in future

The Div Element

  • Were used to divide up content and class or ID tags were used to define content
  • Used for grouping content
  • No special meaning at all
  • Anonymous grouping element

Using WAI-ARIA Roles

  • Web Accessibility Initiative - Accessible Rich Internet Applications
  • More commonly referred to ARIA
  • Landmark role attribute
  • Make part of standard authoring process

Creating Links

Exploring the Anchor Element

  • Href = hypertext reference
  • Target controls where page opens
  • Rel describes relationship between target object and link object
  • Title makes links more accessible which can be read by assistive devices and search engines; always use when creating link to another page or resource

Linking to Pages Within Your Site

  • ../ to go up a directory

Linking to External Pages

  • To link to external site, need to know full url including protocol

Linking to Downloadable Resources

  • Nothing special, browser will download files it can’t render in browser
  • To force download, put download attribute in (boolean)

Linking to Page Regions

  • Fragment identifiers allow us to jump to specific regions within document or even on another document
  • FAQ, glossaries, etc
  • 2 step process
  • If you’re going to give users the ability to jump down, need to give them ability to jump back up
  • When jumping to specific region on different document: /folder#idname
  • When jumping to specific region on external page: look at source code of website to find idname, then tag #idname at end of url
