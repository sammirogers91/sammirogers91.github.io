The basic structure and formatting of an HTML document.

Introducing HTML

  • HTML + CSS + JS = the holy trinity of front-end web development
  • Structure, design, behavior

Basic HTML Syntax

  • Opening and closing tags
  • Elements refer to tag with contents inside
  • Attributes can be specific to tag or global
  • Nesting

The Current State of HTML

  • 1991 - Tim Berners-Lee
  • 1995 - HTML 2.0
  • 1997 - HTML 3.0 and 4.0
  • Currently two specifications of HTML being developed: W3C HTML 5 and WHATWG HTML as a living standard

Basic Page Structure

Exploring an HTML Document

  • HTML document is like a sandwich
  • DOCTYPE says which type of HTML is being used
  • Head and body

DOCTYPE Declarations

  • Supposed to tell browser which version of HTML to parse with
  • Absolutely needed

Understanding Content Models

  • Define the type of content expected inside an element and control syntax rules such as element nesting
  • Block level elements take up their own line
  • Inline elements appear within flow of content itself
  • Metadata: setting up the presentation or behavior of rest of content
  • Embedded content: any content that imports other resources into the document

Formatting Page Content

Formatting Content with HTML

  • Preformatted text tells browser that you want this text to display
  • CSS to make each heading type look how you want

Using Headings

  • Structure pages to determine hierarchy
  • Structuring document, not visual formatting

Formatting Paragraphs

  • Basic, but don’t forget
  • Vertical margins in CSS collapse to avoid double spacing between paragraphs

Controlling Line Breaks

  • <br> doesn't need closing tag
  • Doesn’t create new paragraph

Emphasizing Text

  • Emphasizing/strong =/= italics/bold
  • Emphasizing is functional (screen readers) and italics is just presentational

Displaying Special Characters

  • Named character entities: special characters that would mess up your HTML if put in raw
  • Need to have &___;

Controlling Whitespace

  • Browser is going to ignore every after first space
  • Adding additional whitespace using non breaking space &nbsp;
  • Don’t rely on for visual formatting; instead use CSS
